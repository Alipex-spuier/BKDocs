平台简介

蓝鲸 DevOps 是腾讯蓝鲸研运一体化最佳实践的输出，是面向研发、测试、运维、运营团队的一站式协作平台；整合了腾讯企业级最佳实践，并根据研发场景和能力诉求，形成从需求开始，到代码管理、持续集成、自动化测试，再到应用配置管理、部署交付，运维运营全过程的支持的研运一体化平台。蓝鲸 DevOps 解决方案致力于帮助企业践行 DevOps 文化理念、提升软件研发效能、释放研运业务潜能，为企业数字化转型和竞争力提升提供强大基础能力支撑。

  • 应用场景情况

随着业务快速发展以及市场竞争的加剧，以及新技术、新应用、新产业、新经济、新模式的出现，企业的软件研发的需求越来越多，对快速试错、敏捷协同和持续交付的要求也越来越高。对于企业的软件研发体系和运维体系特别是 IT 部门的产品、开发、测试、质量、运维团队的业务快速交付能力提出了新的要求。原有的开发、运维技术体系难以高效、优质地支撑如此多的业务和应用。因此，建一套具备敏捷协同、持续集成、持续部署的研发、测试、部署一体化平台，实现业务需求快速价值化，提升信息化效率乃至企业的整体竞争力都将有重要的意义。

蓝鲸 DevOps 解决方案以助力业务本身抢得身位优势和具备有效的竞争力为目标。通过蓝鲸 DevOps 平台实现软件研发、测试、运维团队的高效协同业务流端到端的支持及自动化，减少人工成本，提高各部门、团队的持续交付效率和对业务的响应效率；通过顾问咨询服务帮助客户建立涵盖所有过程域的形成统一的、完整的、高效的的研运一体化项目管理流程和规范；采用培训赋能、试点项目推广等方式提升整个团队交付能力，形成持续交付的体系；帮助企业实现持续集成、持续部署全过程的有力支撑，以及对软件的 CI、CD、CO 全生命周期的服务支持。

基于蓝鲸 DevOps 平台来建立研运一体化的支撑体系和管理体系，帮助企业达到研发交付的效率、质量和管控三个方面的整体提升，从而实现业务需求的快速响应助力企业竞争力的提升。

  • 业务应用情况

蓝鲸 DevOps 平台支撑了软件研发业务包括项目管理、需求协同、开发编码、软件测试、质量管理、软件部署等全过程的研运一体化流程。助理整体研发业务响应能力提升，研发交付规范提升，研发交付管控提升，研发交付效率提升，研发交付质量提升,研发交付持续优化改进。

蓝鲸 DevOps 平台的功能包括敏捷协同、代码库、流水线引擎、代码检查、编译加速、自动化测试、质量红线、版本仓库、度量数据等多个能力服务。平台对于企业软件开发采用的技术栈和软件架构等有着最为广泛的支持：支持各种单体架构应用、微服务架构应用、混合架构应用；支持传统部署应用、容器部署应用、混合部署应用；支持网页端、客户端、移动端等多种端，C++、C#、Java、Python、Go、.NET、Object-C、Node.js 等多种主流语言；支持 Windows、Linux、Unix 等多种操作系统；支持分布式、横向扩展、高可用部署等等；支持 7*24 小时不间断服务。通过环境隔离、项目隔离、细粒度权限控制、用户行为审计等范式保障系统的安全性。

经过在不同类型和规模的企业，例如：保险、证券、银行、能源、汽车、航空公司应用，在提高研发效能，提升研发、测试、运维团队协作方面，以及促进研发运维的打通，提升持续交付的效率以及交付稳定性方面，都取得较好的成果。多家企业也借助平台统一了 DevOps 项目的管理规范，提升了业务的竞争力。

项目管理

1. 产品简介

蓝鲸项目管理软件，面向企业和团队进行多级项目管理，满足企业产品线-产品-项目多级项目管理模式；蓝鲸项目管理软件实现了无限层级项目管理、项目规划、无限层级业务需求管理、发布计划、里程碑管理、风险管理、项目服务开关设置等功能，支持需求拆分、基于需求进行发布计划规划管控，支持父子项目间进行版本关联、里程碑关联，监控项目及其下级项目风险，帮助管理者管控项目整体进度有序进行，支持直观展示项目各里程碑的进展情况，管理项目各阶段的交付物等。

蓝鲸项目管理软件实现对多个管理项目的集中管理和协调管理。项目管理包括了项目概览、项目计划、项目资源、任务、交付物、里程碑、风险监控、项目流程、文档功能；支持设置项目计划和资源，指定项目任务，以及控制项目过程的流程、交付物和存在的风险，支持项目的协同管理和文档的知识共享；同时对需求进行管理，实现需求的提出确认、需求变更管理、需求版本管理,自上而下整体规划和需求拆分、可视化监控项目状况和从下至上的进度反馈，支撑资源合理分派，有效对项目全过程进行度量分析，帮助企业提升团队效率。

2. 核心功能

  • 基本信息： 支持查看并修改基础信息、新建、收藏、编辑、添加子项目、归档项目、删除项目、初始化示例数据等功能；

  • 需求管理： 用于管理需求，包括需求的查询、创建、编辑、归档、分派、流转、删除等操作，可以支持需求之间存在的层级关系和使用动态信息，支持在项目研发过程中针对用户需求的管理活动。支持项目任务进度、团队沟通与协作，集成看板管理、需求管理、版本管理、迭代管理、任务管理、功能日志等功能，并支持工作项类型、工作项字段、工作项页面、工作流、消息通知及插件扩展的自定义；

  • 版本管理： 支持对项目版本进行查看/筛选、创建、编辑、规划、关联、完成、删除等功能；

  • 里程碑： 里程碑通过控制项目工作进展以保证实现总目标，支持创建、管理交付物、关联里程碑、编辑、删除等功能；

  • 设置： 支持服务启停、需求设置、业务模型设置、权限设置等功能，具备自定义需求类型、字段、工作流、业务模型等能力，满足各种使用场景的需要。

3. 产品优势

  • 提升效能： 量身打造的产品研发生命周期解决方案，帮助团队解决项目管理过程问题，提升研发效能；

  • 灵活配置： 企业既可以统一设置需求管理模型，也可以根据团队和项目进行个性化定义，满足企业不同时期、不同阶段的协同与需求管理诉求，贴合企业演进需要。

敏捷协同

1. 产品简介

蓝鲸敏捷协同软件，面向企业敏捷开发过程的需求管理、任务分配、迭代规划、版本设计等方面，支持基于项目级的工作项类型、工作项字段、创建页面、工作流、标签等实体模型自定义，实现了 backlog 管理、看板管理、工作项、版本管理、迭代管理、报表中心、个人代办、工作日历、活动追踪等功能，满足用户自定义 Scrum、Kanban 等不同的面板视图，并提供以工作项为中心提供第三方插件支持，实现敏捷开发中“需求—设计—开发—测试”的全程信息掌控，同时集成了 git、邮件、微信等多种通知方式，为研发团队工作开展提供信息共享与协同能力服务，解决团队存在的沟通壁垒、效率竖井，价值流动阻塞等问题，通过看板视图等让管理者快速了解项目整体状况，支撑研发效能度量。

2. 核心功能

  • 项目类型： 支持常规项目类型、敏捷项目类型。常规项目类型可以进行需求、任务和缺陷的管理。敏捷项目类型可以进行规划并跟进迭代，支持适用于定期交付的团队。能够有效支撑企业传统、敏捷双态项目模式，帮助企业逐步从传统向敏捷模式过渡；

  • 看板： 支持提供两种类型的面板：Kanban 面板和 Scrum 面板。Kanban 侧重于将工作流程和制品可视化，以促进增量改进现有的流程，可以帮助项目优化开发流程，更快地（每天甚至更短时间）集成和交付给用户使用。Scrum 方法建议将开发过程按迭代周期管理，每个迭代（建议 1-4 周）结束后集成和交付用户使用，侧重于计划、执行、交付命名为 Sprint 的时间驱动的工作任务；

  • 工作项： 工作项管理功能，包括查询（支持搜索查询、多条件组合筛选）、创建、修改、删除、选择类型（Epic、故事、任务等）、关联工作项（关联缺陷、父子关系等）、关注、状态流转、评论、查看变更记录和操作日志、关联或解绑代码库、关联流水线、查看代码提交记录、导入、导出等。同时支持可以根据需要设置优先级、标签、经办人、测试用例、衍生图等，全过程追踪工作项进度，提高工作效率；

  • 工作日历： 工作日历菜单中可以选择按日、月 、年 查看工作项基本信息；

  • 活动： 活动记录当前项目下的所有操作日志，可以通过筛选查找对应活动；

  • 统计报表： 统计报表用于分析当前工作项数据并以图标方式直观展示，支持工作项分布/趋势/缺陷回归/阶段/响应时间/等，可以进行故事周期统计/迭代分析/工时分析/缺陷密度统计等，支持工作项自定义字段统计，利于工作开展，沉淀项目知识，提升团队协作效率；

  • 版本管理： 版本管理包含有查看（支持搜索查询、多条件筛选）、创建、规划（支持一次性批量规划）、编辑、发布、删除等，能够对产品生命周期进行管理，支持版本与迭代建立关联关系；

  • 迭代管理： 基于 Scrum 模式，提供迭代的任务列表模式管理。包含有查看（支持搜索查询、多条件筛选）、创建、规划（支持一次性批量规划）、编辑、删除等操作。通过管理任务列表及迭代状态，实现项目任务的计划、执行可视化管理；

  • 任务管理： 任务管理支持项目过程阶段相关活动的管理,实现研发任务管理与跟踪，可以通过自定义工作流和工作节点管理研发任务的发起、评审、拆分，任务与版本、迭代和其它 issue 如需求、故事、epic 等的关联，任务内容支持自定义如迭代周期、任务内容、任务状态、任务负责人、任务时间等，支持内容的可视化管理，实现任务确认、开发、测试、 上线、验收等状态的跟踪管理。任务可以与迭代和版本进行关联。任务也可以与开发信息如代码库、流水线、测试用例进行关联，实现了项目任务关联和进度的管理跟进；

  • 缺陷管理： 支持研发过程中测试发现缺陷的登记、跟踪、验证、需求关联等缺陷生命周期的闭环管理，可以指定经办人，优先级，添加标签，支持富文本编辑和附件上传等；

  • 知识库： 知识库包含文件管理和 Wiki 在线文档，两者均可通过全文搜索快速定位至所需文档。其中的文件管理功能是用来存储项目上的相关文档，使用文件夹以进行文件管理，具备（txt、Excel、Word、gif、pdf 等常见）文件在线预览功能；Wiki 在线文档便于成员协作和更新文档，使用列表方式对文档进行管理，支持富文本和 Markdown 格式；

  • 报告管理： 报告管理包含创建项目报告发送给项目成员以及接受其余项目成员发送的报告，并且项目报告支持关联迭代，方便所有团队成员关注开发质量；

  • 设置： 设置不仅包含展示当前项目的相关信息并支持修改，功能开关，自定义设置工作项类型、字段、标签、模板、页面、工作流等功能，具有强大的配置灵活性和扩展性，而且支持消息通知、用户权限、模板管理、插件中心、WebHook 等功能，能够满足各种管理需要。

支持多种敏捷研发实践的协作平台，管理故事、任务、缺陷、版本、迭代、里程碑等；强大的配置灵活性和扩展性，支持各种管理需要。

  • 双态管理： 支持协同优化提升，可以根据企业和团队的特点，选择相应研发模式，并逐步向规模化敏捷演进，实现企业项目与需求管理的高效开展；

文档管理

1. 产品简介

蓝鲸文档管理软件，面向企业及团队进行知识共享和文档管理，实现了文档统一管理和基于富文本、Markdown 格式进行在线文档 Wiki 编辑的协作方式；蓝鲸文档管理软件实现了对项目过程中产生的架构设计、操作手册、产品说明等文档材料进行管理。文档管理作为一款企业知识管理应用，是知识沉淀、协作和共享的统一平台；同时支持组织知识内容结构，便于管理、检索查阅等。通过细化到文件级的操作权限控制，还可以对其中的资源单独授权。

文档管理服务下，每个文库作为独立单元，授权与数据完全独立。租户管理员可以在该租户下创建文库，并且对每个文库单独授权，除此之外，还可以指定文库管理员来管理文库。

文库中提供文件管理以及 WIKI 管理，其中，如果有需要甚至可以对于每个文件或 WIKI 单独授权，可以控制对资源的查看、下载、编辑、上传、删除等一种或多种权限。

2. 核心功能

数据工厂

1. 产品简介

测试数据准备是一项繁杂且成本高的问题，通过数据工厂，可以快速完成测试数据准备工作。数据工厂根据数据生成策略自动生成测试数据，支持关系型数据库和 Redis 等，按规则生成、卸载测试数据。将数据工厂与同为蓝鲸体系的锋测软件结合使用，可以极大的减轻测试人员准备测试数据及恢复数据状态的工作，提升软件测试的效率。

2. 核心功能

任务中心

  • 数据库任务： 数据库任务是对接口、字典、数据源、数据文件等基础数据的组装，分为生成任务和卸载任务。生成任务：是在任务执行前将组装的数据初始化到数据库，生成任务分为接口生成、数据文件注入、随机生成和 SQL 生成四种类型。卸载任务是在任务执行后对数据库数据进行卸载，卸载任务分为接口卸载、SQL 卸载和条件删除三种；

  • Redis 任务： Redis 数据源任务是对接口、字典数据的组装，分为生成任务和卸载任务两种。生成任务：是在任务执行前将组装的数据初始化到数据库，生成任务分为接口生成和字典注入两种类型。卸载任务是在任务执行后对数据库数据进行卸载，卸载任务分为接口卸载和字典卸载两种；

  • 任务组： 任务组是任务的编排与组合，可以编排组合生成任务和卸载任务，为了减轻服务器的压力，组合的生成任务和卸载任务分别最多只能有 5 个，任务组包含新增、编辑、快速关联生成和卸载任务、清空任务、调试等功能；

数据管理

  • 任务接口： 任务接口管理主要是用来管理数据库任务所需要使用的接口，支持添加、管理、筛选等操作；

  • 数据源管理： 数据源管理为数据库任务提供了数据源，包含关系型数据源和 Redis 数据源，支持添加、管理、筛选等操作；

  • 字典管理： 字典管理是基础数据管理，为数据管理和数据文件中的键值对引用提供管理功能，以键值对的形式存储。支持添加、管理、筛选、导入、导出等操作；

  • 数据文件管理： 数据文件管理通过 csv 文件的形式管理参数的批量值，支持添加、管理、筛选等操作；

  • 环境管理： 对于接口自动化测试，当用户采用手工测试方式时，需要设置接口所在的环境才能进行测试。通过环境设置，可以指定被测接口所在的环境，还可以对环境编辑、删除和查看详情。支持添加、管理等操作；

  • 凭证管理： 对于接口自动化测试（如果接口需要访问凭证），当用户采用手工测试方式时，需要设置凭证才能进行测试。通过凭证设置，可以新增手工接口测试需要的凭证，凭证有 4 种类型，分别是 Token、basic auth、user credential、Oauth2.0。支持添加、编辑、删除、查看凭证详情等操作。

3. 产品优势

  • 高性能： 通过配置数据库字段数据生成规则，即可轻松生成最高百万级数据；

  • 灵活性： 灵活的编排数据生成和卸载任务，满足不同场景数据生成和卸载要求；

  • 与锋测集成： 通过与锋测集成，大幅度减轻测试人员在测试前数据及测试后卸载数据的工作量；

  • 智能卸载： 可智能识别数据库在测试执行前后数据的变化，轻松恢复数据库数据到测试执行前的状态。

质量红线

1. 产品简介

将质量管理流程和质量控制指标结合到质量管理工具平台，实现自动化检测、是提升研发效能的重要途经，质量红线通过设置质量标准，控制流水线的行为，使得其产出物必须符合质量标准，支持建立自定义的质量基线或准入门禁，自动收集内外部的质量数据，根据质量拦截规则自动拦截流水线，确保交付的质量，质量人员只需要关注在设定质量标准。

2. 核心功能

总览

质量门禁实施情况的概要展示，包括：规则数、指标数、生效流水线数、红线拦截次数、流水线拦截 Top5、生效流水线执行/拦截趋势，拦截历史最新 10 条；

红线规则

红线规则涵盖创建规则以及规则操作两个功能。根据门禁项控制需要创建红线规则，包括指标、控制点、控制指标项的设定，规则生效的流水线范围，以及规则触发后续处理操作设置。规则操作包括规则的启用、停用、编辑、删除，详情查看等；

指标列表

支持查看、管理多项指标的相关信息，包括全部指标、脚本任务指标、研发商店指标和系统指标等，可按要求填写相关信息，自定义脚本任务指标；

通知组

通知组用于人员分组，为新增或者编辑拦截规则时提供组别选择；

流水线联动

将质量红线插入流水线中是对于质量红线最主要的应用，通过插入指定的监测点，可以检查指定的指标是否达到最低标准。

3. 产品优势

  • 拦截展示： 支持质量红线实时情况的概要展示，包括规则数、指标数、生效流水线数、红线拦截次数、流水线拦截 Top5、生效流水线执行/拦截趋势，拦截历史最新 10 条等；

  • 提供系统中内置的指标： 系统中内置代码扫描的指标项，支持代码扫描结果与质量红线联动，通过扫描结果由质量红线控制流水线执行；

  • 支持不同控制项创建规则： 支持根据门禁项控制需要创建红线规则，包括控制点、控制指标项的设定，规则生效的流水线范围，以及规则触发后续处理操作设置；

  • 支持灵活扩展指标： 支持灵活扩展指标，用户可根据需要灵活扩展指标，提供 Bash 指标以及研发商店指标扩展、指标项定义后通过流水线的相关插件获取指标值，触发指标判断；

  • 多支持通知方式： 流水线执行完质量红线原子，可以通知发送至规则设置时相应的接收人员，包括邮箱、审核人等。

管控平台

1. 产品介绍

蓝鲸管控平台，是整个蓝鲸平台的底层管控系统，是蓝鲸所有其他服务的基础，是蓝鲸服务体系与用户机器的连接器。

蓝鲸管控平台是典型的两层分布式 C/S 结构，主要包含智能 Agent，提供各种服务的 Server，以及 zookeeper、redis、MySQL 等周边保障模块。其中 Agent 是部署在业务机器上的程序，每台业务机器理论上只可以部署一个；其他模块部署无具体要求，用户可以单独部署，也可以混合部署。

在整个蓝鲸体系中，唯独蓝鲸管控平台没有直面用户，但蓝鲸管控平台在蓝鲸体系中却是不可或缺的，它为蓝鲸其他平台提供了人机交互的通道与能力。蓝鲸管控平台主要提供了三种类型的服务能力：文件分发传输能力、命令实时执行与反馈的能力、大数据采集与传输的能力。

2. 核心功能

文件分发与传输

文件分发是指用户从指定机器将指定文件批量传输到特定范围的机器上。文件分发是用户的一种行为，传输则是针对用户行为的程序行为；

实时任务执行

  • 命令类型——linux 支持 bash 命令、windows 支持 cmd 命令、AIX 支持 ksh 命令，支持各种自定义可执行文件格式程序的启动，支持各种解释性语言程序的执行；
  • 脚本类型——linux 支持 shell 脚本、windows 支持 bat 脚本（安装有 cygwin 的额外支持 shell 脚本）、powershell、AIX 支持 ksh 脚本，以及各种系统支持的解释性脚本程序;

数据采集与传输

Agent 开放数据发送接口、cmdline 及 SDK，供用户开发自定义的数据采集程序或脚本；

Agent 支持采集器插件化，自动加载采集插件，并监控插件的存活状况。如果采集插件异常终止，则重新拉起采集插件；如果多次拉起失败则告警，支持实时数据快照，支持动态负载均衡；

集群管理

支持自动发现，管控平台同一个集群内的模块均支持自动发现，用户扩缩容任何节点，系统均能实时感知，并调整通讯策略，保证服务的高可用，支持按照 Agent 链接数进行负载均衡，可以进行 Agent 状态查询和多区域负载均衡。

3. 产品优势

  • BK Agent，强大的执行代理： 蓝鲸智能 Agent 程序，可以安装在业务需要管控的实体机、虚拟机或者容器里面，蓝鲸 Agent 是蓝鲸管控平台提供三大服务能力（命令执行、文件传输、数据上报）的实际执行者，蓝鲸 Agent 所在机器的通讯策略、网络状况需要在安装前调整好才能发挥其所有能力。

  • BK TaskServer，提供海量管控能力： 蓝鲸管控平台任务及控制服务端程序，该程序提供对集群内 Agent 的管理能力，并支持对 Agent 批量下发执行发命令或脚本，最快任务流转时间小于 2s。

  • BK FileServer，基于 BT 协议，传输更高效： 管控平台文件传输控制服务端程序，该程序对指定范围内 Agent 节点提供 BT 种子服务，保证对传输的安全性、不同区域及业务模块间的隔离性，并控制 BT 传输在有限的贪婪特性范围内。单独部署 BK FileServer 并不能提供文件传输服务，受限于安全性考虑，BK FileServer 必须和 BK TaskServer 配合才能完成完整的文件分发流程。

  • BK DataServer，海量数据处理不再是瓶颈： 蓝鲸管控平台数据传输服务端程序。该服务端主要提供对 Agent 采集的数据进行汇聚、分类、流转能力。对于普通的千兆网卡机器，BK DataServer 能够最大提供 100MB/s 的数据处理能力。BK DataServer 可以单独为用户提供数据服务，而不需要其他服务端程序配合。

应用配置管理

1. 产品介绍

应用配置管理软件是一款在分布式环境中对应用配置进行集中管理和推送的产品，能够集中化管理不同环境、不同集群的配置。凭借配置变更、配置推送、历史版本管理、配置变更审计等配置管理工具，帮助用户集中管理所有应用环境中的配置，降低分布式系统中管理配置的成本，并降低因错误的配置变更造成可用性下降甚至发生故障的风险，为应用部署发布提供基础。

2. 核心功能

节点管理

节点管理包含节点、挂载主机、挂载配置、挂载参数管理，支持查询、创建、编辑、删除、批量操作等；

配置管理

配置管理包含配置、用户配置、历史配置管理。支持展示配置信息、用户版本配置信息、数字版本配置信息等，具备回滚版本、对比版本的能力，支持批量操作等；

标签管理

支持查看标签的名称、是否关联、创建时间、修改时间，可以进行新建、编辑、多条件筛选、批量删除标签等操作；

模板管理

支持 properties、json、xml、yml、yaml、txt 等格式的配置文件模板。可以进行新建、编辑、删除、批量删除、多条件筛选、批量导入导出、查看详情、配置模板等操作。

3. 产品优势

  • 个性化应用结构层级定义： 可将应用按环境、集群、服务三个层级维度进行个性化划分和管理；

  • 支持参数化应用配置管理： 可在应用配置信息中使用参数占位符，实现配置信息参数模板化；

  • 精细化权限控制： 与权限中心集成，实现对配置信息管理的精细化权限控制；

  • 支持应用配置的快速复制： 可将已有的配置信息进行复制，实现配置信息的快速创建和更新；

  • 支持可视化的历史版本信息对比： 用户可选择两个历史版本进行对比配置信息差异；

  • 支持项目范围内的多应用管理： 在项目范围下可进行多应用定义与管理；

  • 支持联动 CMDB： 联动 CMDB，导入主机信息；

  • 支持区分应用级配置和主机级配置： 可以针对应用或主机生成不同配置文件；

  • 分发： 借助蓝鲸的管控平台下发配置文件。

环境管理

1. 产品介绍

环境管理提供对构建环境及单个构建节点的管理，在目标构建机任意路径下执行环境管理提供的脚本命令即可完成构建机节点的导入。

2. 核心功能

节点管理

构建节点用来完成构建任务的构建机，用于完成源代码拉取、编译和打包功能。蓝鲸 DevOps 平台支持导入 CMDB 和自定义构建机作为环境节点。导入的 CMDB 作为部署机器，导入第三方构建机可以在编排流水线时作为构建资源使用。支持 MacOS、Linux、Windows 等第三方构件机的快速接入，支持从 CMDB 中导入应用服务器；

环境管理

环境由一组节点组成，构建环境可以导入自定义构建机，构建环境可以在流水线中使用。部署环境只能选择同一业务下的节点，部署环境可以在企业部署插件中使用。支持对导入的第三方构建机、应用服务器二次定义环境分组，支持环境中节点的自定义增减，支持选定环境的配置项自定义设置等。

3. 产品优势

  • 多操作系统支持： 支持 MacOS、Linux、Windows 等多种操作系统及版本的支持；

  • 接入快捷灵活： 对于 MacOS、Linux 的第三方构建机，通过命令执行即可完成接入。Windows 构建机，通过安装 Agent 即可完成接入。

作业平台

1. 产品介绍

作业平台（JOB）是一套基于蓝鲸智云管控平台 Agent 管道之上的基础操作平台，具备万级并发处理能力。除了支持脚本执行、文件拉取 / 分发、定时任务等一系列可实现的基础运维场景以外，还运用流程化的理念很好的将零碎的单个任务组装成一个作业流程。而每个任务都可做为一个原子节点，提供给其它系统和平台调度，实现调度自动化。

2. 核心功能

首页

作业平台提供了作业总览视图，通过总览视图能够直观明了的查看当前 IT 环境中的业务主机数，常用作业数，近 30 天的任务数，定时任务数，历史任务执行情况，任务执行时长统计，最近任务执行记录等。IT 运维人员能够做到对于基础作业情况一目了然；

作业执行

  • 快速脚本执行： 可通过手动编写、从已有的脚本克隆、从本地上传、从公共脚本选择四种方式导入脚本并批量执行；

  • 快速分发文件： 支持本地上传和服务器上选择两种方式，除了一对多，还可以多对多、多对一形式进行文件分发；

  • 快速执行 SQL 脚本： 可通过手动编写、从已有的脚本克隆、从本地上传方式导入脚本并批量执行；

  • 常用作业执行： 查看已经保存的作业任务，可进行 “立即执行、定时启动、编辑、克隆、删除” 操作；

  • 新建作业： 将多个脚本执行或文件分发的节点结合在一个流程步骤内，形成一个新建的作业任务；

  • 定时作业： 对已创建的作业任务进行定时启停，可修改和删除；

业务管理

  • 脚本管理： 支持对作业下的脚本进行查询、新建、执行、编辑和删除等操作；

  • SQL 管理： 支持对作业下的 SQL 脚本进行查询、新建、执行、编辑和删除等操作；

执行历史

对所有在作业平台执行过的脚本执行、文件分发、API 调用、定时任务等操作进行记录，方便查看详情；

个人设置

  • 通知管理： 针对作业执行成功、失败、等待执行等三类场景进行通知，确保完成整个作业执行流程；

  • 个人设置： 支持作业执行的一些默认设置、安全设置和作业迁移服务的迁入和迁出等；

管理员入口

  • 公共脚本： 支持对公共脚本的查询、新建、执行、编辑和删除等操作；

  • 账户管理： 对服务器上的账户和针对 SQL 执行功能的 DB 账户（仅企业版支持）进行管理如查询、添加、编辑和删除；

  • IP 白名单： 支持 IP 白名单的查询、新建、编辑和删除。

3. 产品优势

支持海量脚本执行、文件分发、 SQL 执行、 灵活 作业编排，实现企业各种复杂运维场景的一键式、自动化运维操作平台。

基于时间窗口的发布

    5. 默认：即部署所有服务器；指定：即选择部署部分服务器；分批：即按批次分批执行、适用于环境或发布版本包有相互依赖的场景，且每个批次也支持单独部署，如：只部署第一、第三批次的设备或只部署二、四批次的设备

  • 部署与发布策略： 支持采用定期部署策略，具备按天、小时进行部署的能力, 支持应用和配置 (的部署分离)进行分离；

  • 发布过程可视化： 支持流水线可视化，交付过程组织内部可见，团队共享度量指标。

  • 体系化： 面向度量统计、问题分析、优化跟踪提供体系化的服务能力，助力企业持续反馈与优化机制建立。
